# Contributing to PromptDeck

Thanks for your interest in contributing to PromptDeck!

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your feature or fix
3. **Make your changes** — keep the single-file architecture (everything in `index.html`)
4. **Test** by opening `index.html` in a browser and verifying your changes work
5. **Submit a Pull Request** with a clear description of what you changed and why

## Guidelines

- **Keep it simple.** PromptDeck is a single-file app by design. Don't add build tools, frameworks, or external dependencies unless absolutely necessary.
- **Test in both Electron and browser.** The app should work in both environments.
- **Match the existing style.** Use the same CSS variable patterns, naming conventions, and code structure.
- **No telemetry or analytics.** PromptDeck is private by design. Don't add tracking.

## Reporting Issues

Open an issue on GitHub with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Your OS and browser/Electron version

## Feature Ideas

Open an issue tagged with `enhancement` and describe:
- What the feature does
- Why it's useful
- How it fits with the existing app

We appreciate all contributions — from bug fixes to new features to documentation improvements.
